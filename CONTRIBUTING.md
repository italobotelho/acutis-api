# Contributing to The Acutis API

First off, thank you for considering contributing! This project relies on the global Catholic and developer community to grow.

## 1. Branch Naming Convention
Before you start coding, create a branch with a descriptive name based on the type of work:
- `feature/` for new API endpoints or scraper logic (e.g., `feature/miracles-endpoint`)
- `data/` for database inserts (e.g., `data/add-st-thomas-aquinas`)
- `bugfix/` for resolving errors
- `docs/` for documentation updates

## 2. Commit Message Standard
We follow [Conventional Commits](https://www.conventionalcommits.org/). Your commit message should look like this:
- `feat: add GET /patronages endpoint`
- `fix: correct birth date of St. Francis`
- `docs: update README examples`

## 3. How to Submit a Pull Request (PR)
1. Fork this repository.
2. Create your branch from `main`.
3. Make your changes and commit them.
4. Push to your fork and submit a Pull Request.
5. Wait for the core team to review your data for historical accuracy and code quality.
